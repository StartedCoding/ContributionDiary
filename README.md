# 🧠 ContributionDiary

> A structured, open-source system to log developer contributions, learning, and daily progress.

---

## 📖 Purpose

ContributionDiary is designed to help developers **track what they build, learn, and struggle with** — in a structured and public way.

This is not just a diary.  
It’s a **system for consistency, reflection, and growth**.

---

## 🚀 What You Can Do Here

- 🧩 Log open-source contributions (PRs, bug fixes, features)
- 🧠 Track what you learn daily
- ⚙️ Document problems and how you solved them
- ⏱️ Maintain hourly productivity logs
- 🔥 Build consistency through daily entries

---

## 📂 Project Structure
```ts
/entries
/your-name
YYYY-MM-DD-project-name.md

/templates
contribution-template.md
hourly-log-template.md
```

---

## 🧩 Types of Logs

### 1. 📌 Contribution Log (Deep Work)

Used for meaningful work like:
- Open-source contributions
- Features built
- Bugs fixed

#### Example:

📌 Contribution Entry
##### 🧩 Project

> StartedCoding Blog Platform

##### 🚀 Contribution

> Implemented rate limiting middleware

##### 🧠 What I Learned

> How token bucket algorithm works

##### ⚙️ Challenges Faced

> Handling burst traffic

##### 🛠️ How I Solved Them

> Used in-memory store with refill logic

##### ⏱️ Time Spent

> 3 hours

##### 🪶 Tags

> #backend #system-design


---

### 2. ⏱️ Hourly Log (Consistency Tracking)

Used to track daily productivity.

#### Example:

##### ⏱️ Hourly Log
```md
> 09:00 - 10:00 → Studied Graph Algorithms
> 10:00 - 11:00 → Solved 2 problems
> 11:00 - 12:00 → Worked on MERN project
```
---

## 📝 Templates

Use templates from `/templates` folder:

- `contribution-template.md`
- `hourly-log-template.md`

---

## 👥 How to Contribute

Want to log your journey publicly? Follow these steps:

1. Fork this repository  
2. Create a folder inside `/entries/your-name/`  
3. Copy a template from `/templates`  
4. Add your entry  
5. Create a Pull Request 🚀  

---

## 🔥 Consistency > Intensity

Even small daily logs compound into massive growth over time.

Start logging. Stay consistent. Build proof of work.
